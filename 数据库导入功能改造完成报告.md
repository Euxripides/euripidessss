## 3. 新增功能

- 新增“数据库导入”入口。
- 支持新建、编辑、删除、测试数据库连接。
- 支持数据库、schema、表浏览。
- 支持表结构读取、分页预览、搜索。
- 支持默认只读 SELECT 查询。
- 支持安全新增、修改、删除接口和前端操作页。
- 支持字段自动映射、强制确认、保存和复用。
- 支持创建数据库导入任务并生成 Flow 可用的 `flow_sessions` 数据集。

## 4. MySQL 支持情况

- 使用 `github.com/go-sql-driver/mysql`。
- 支持默认端口 3306、连接测试、库列表、表列表、字段读取、分页 `limit/offset`、参数占位符 `?`。
- 标识符使用反引号转义。

## 5. PostgreSQL 支持情况

- 使用 `github.com/lib/pq`。
- 支持默认端口 5432、连接测试、数据库列表、schema 列表、表列表、字段读取、分页 `limit/offset`、参数占位符 `$1`。
- 默认过滤 `pg_catalog`、`information_schema`、`pg_toast%`。
- 标识符使用双引号转义。

## 6. 字段映射实现

- 后端按字段名、大小写归一、空格/下划线归一和常见中文/英文别名评分。
- 必填字段缺失或低置信度时，前端要求用户确认后才能导入。
- 映射规则按连接、数据库、schema、表、字段 hash 保存，避免影响无关表。

## 7. 大数据导入实现

- 表预览强制分页，默认 100 行，最大 1000 行。
- 导入任务按页读取，默认单表最多导入 100000 行，避免一次性载入全表。
- 导入进度按页写回任务状态，错误行写入任务错误列表。
- 支持 cancel 状态检查；前端第一版仍等待 start 请求完成。

## 8. 性能优化

- 前端表格使用分页和虚拟滚动。
- 后端配置连接池最大连接数、空闲连接数和连接生命周期。
- SQL 查询设置请求级超时。
- 搜索和筛选使用参数化查询，避免大 JSON 一次性返回。

## 9. 安全措施

- 密码仅在用户勾选保存密码时持久化。
- 本地配置写入 `backend/data/db_import/db_import_config.enc`，使用 AES-GCM 加密，目录已加入 `.gitignore`。
- 日志和 API 响应不返回密码。
- 表名、字段名只作为经过转义的标识符拼接，筛选值使用参数化查询。
- 默认 SQL 查询只允许 SELECT/WITH。
- 无条件 UPDATE/DELETE 会被后端拒绝。
- 删除操作前端有二次确认。

## 12. 使用说明

1. 打开“资金流向图”。
2. 点击选择来源数据。
3. 选择“数据库导入”。
4. 新建并测试 MySQL/PostgreSQL 连接。
5. 打开连接，选择数据库、schema、表。
6. 在“表数据/表结构/查询”中确认数据。
7. 在“字段映射”中确认必填字段并保存。
8. 点击“导入流向图”，导入完成后使用现有字段筛选和生成图谱流程。
